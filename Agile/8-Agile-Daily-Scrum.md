# โครงร่างคู่มือ Daily Scrum (สองภาษา)

## **1. บทนำ / Introduction**

**ไทย:**  
บทนำนี้อธิบายวัตถุประสงค์และหลักการพื้นฐานของ Daily Scrum ซึ่งเป็นกิจกรรมสำคัญในกรอบงาน Scrum เพื่อส่งเสริมการสื่อสาร โปร่งใส และการปรับตัวภายในทีมพัฒนา

**English:**  
This introduction outlines the purpose and basic principles of the Daily Scrum, a key event in the Scrum framework designed to promote communication, transparency, and adaptation within the Development Team.

## **2. นิยามและวัตถุประสงค์ / Definition & Purpose**

**ไทย:**  
Daily Scrum คือกิจกรรมที่กำหนดเวลา 15 นาทีสำหรับทีมพัฒนาเพื่อประสานงานและวางแผนงานสำหรับ 24 ชั่วโมงถัดไป โดยมีวัตถุประสงค์เพื่อตรวจสอบความคืบหน้าและระบุอุปสรรค

**English:**  
The Daily Scrum is a 15-minute time-boxed event for the Development Team to synchronize activities and create a plan for the next 24 hours, aiming to inspect progress and identify impediments.

## **3. หัวข้อหลัก / Key Topics**

### **3.1 โครงสร้างและรูปแบบ / Structure & Format**
**ไทย:**
- เวลา: ไม่เกิน 15 นาที
- ความถี่: ทุกวันทำงาน
- สถานที่: คงที่และสม่ำเสมอ
- รูปแบบ: ไม่เป็นทางการ มักยืนประชุม

**English:**
- Time: Maximum 15 minutes
- Frequency: Every working day
- Location: Consistent and fixed
- Format: Informal, often standing

### **3.2 คำถามหลัก 3 ข้อ / 3 Key Questions**
**ไทย:**
1. มีอะไรที่ทำเสร็จสมบูรณ์นับตั้งแต่ Daily Scrum ครั้งล่าสุด?
2. มีอะไรที่คาดว่าจะทำเสร็จสมบูรณ์ก่อน Daily Scrum ครั้งต่อไป?
3. มีอุปสรรคอะไรที่ขัดขวางการทำงาน?

**English:**
1. What did I complete since the last Daily Scrum?
2. What do I plan to complete before the next Daily Scrum?
3. What impediments are blocking my progress?

### **3.3 บทบาทและหน้าที่ / Roles & Responsibilities**
**ไทย:**
- สมาชิกทีมพัฒนา: แบ่งปันความคืบหน้าและอุปสรรค
- Scrum Master: ช่วยขจัดอุปสรรคและรักษาเวลา
- Product Owner: เข้าร่วมฟังได้แต่ไม่จำเป็นต้องพูด

**English:**
- Development Team Members: Share progress and impediments
- Scrum Master: Facilitates and removes impediments
- Product Owner: May attend as a listener

## **4. การออกแบบคู่มือ / Manual Design**

**แนวคิดการออกแบบ:**
- รูปแบบพกพา ขนาด A5 หรือดิจิทัล PDF
- ใช้สีที่แตกต่างสำหรับแต่ละภาษา
- ไอคอนประกอบการสื่อสารที่ชัดเจน
- โครงสร้างแบบเป็นขั้นตอน (step-by-step)

## **5. Workflow Daily Scrum**

```
1. เริ่มตรงเวลา → 2. Scrum Master เปิดการประชุม → 3. สมาชิกพูดตอบคำถาม 3 ข้อ → 4. อภิปรายปัญหา (หากจำเป็น) → 5. จบภายใน 15 นาที → 6. ลงมือทำงานตามแผน
```

## **7. CHECKLIST Template**

### **Checklist ภาษาไทย:**
- [x] ประชุมตรงเวลา
- [x] ทุกคนพร้อมประชุม
- [x] จำกัดเวลา 15 นาที
- [x] ตอบคำถาม 3 ข้อครบ
- [x] บันทึกอุปสรรค
- [x] มีแผนชัดเจนหลังประชุม

### **Checklist English:**
- [ ] Meeting starts on time
- [ ] All members present
- [ ] Time-boxed to 15 minutes
- [ ] All 3 questions addressed
- [ ] Impediments documented
- [ ] Clear plan established

## **10. ตัวอย่างบทสนทนา / Conversation Examples**

**ไทย:**
"1.เมื่อวานฉันทำการทดสอบโมดูลการล็อกอินเสร็จเรียบร้อย 2.วันนี้จะเริ่มพัฒนาหน้าสรุปรายงาน และ 3.มีอุปสรรคเรื่องการเชื่อมต่อฐานข้อมูลที่ต้องขอความช่วยเหลือ"

**English:**
"1.Yesterday I completed testing the login module. 2.Today I'll start developing the report summary page. 3.I'm facing a database connection issue that needs assistance."
