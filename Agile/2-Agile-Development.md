# **คู่มือการใช้งาน Scrum แบบครบวงจร (Thai-English Bilingual)**

## **1. บทนำ / Introduction**

### **English Version**
```
INTRODUCTION TO SCRUM
=====================

Scrum is an Agile framework designed to help teams work together to 
deliver value in an iterative and incremental manner. Originally 
developed for software projects, Scrum has been successfully adapted 
to various industries due to its flexibility and focus on continuous 
improvement.

This guide provides a comprehensive overview of Scrum fundamentals, 
including roles, events, artifacts, and practical templates for 
immediate implementation.
```

### **ภาษาไทย**
```
บทนำสู่สกรัม
=============

สกรัมคือเฟรมเวิร์กแบบ Agile ที่ออกแบบมาเพื่อช่วยให้ทีมทำงานร่วมกัน 
ในการส่งมอบคุณค่าแบบเป็นขั้นตอนและเพิ่มพูนอย่างต่อเนื่อง เริ่มแรกพัฒนา
สำหรับโครงการซอฟต์แวร์ ปัจจุบันสกรัมถูกนำไปใช้กับอุตสาหกรรมต่าง ๆ 
เนื่องจากมีความยืดหยุ่นและเน้นการปรับปรุงอย่างต่อเนื่อง

คู่มือนี้ให้ภาพรวมพื้นฐานของสกรัมอย่างครบถ้วน รวมถึงบทบาท กิจกรรม 
สิ่งส่งมอบ และเทมเพลตสำหรับการใช้งานได้ทันที
```

## **2. นิยาม / Definition**

### **English Version**
```
DEFINITION
==========

Scrum (n): A lightweight framework that helps people, teams, and 
organizations generate value through adaptive solutions for complex 
problems.

Key Characteristics:
• Lightweight - Minimal overhead
• Simple to understand - Easy to grasp concepts
• Difficult to master - Requires practice and commitment

Empirical Process Control (3 Pillars):
1. Transparency - All aspects visible to responsible parties
2. Inspection - Frequent checks on progress
3. Adaptation - Adjustments based on inspection
```

### **ภาษาไทย**
```
นิยาม
=====

สกรัม (คํานาม): แบบแผนการทำงานแบบเบาที่ช่วยให้บุคคล ทีม และองค์กร
สร้างคุณค่าผ่านทางแก้ไขปัญหาที่ซับซ้อนได้อย่างปรับตัวได้

คุณลักษณะสำคัญ:
• เบา - มีขั้นตอนน้อย
• เข้าใจง่าย - แนวคิดไม่ซับซ้อน
• ยากที่จะเชี่ยวชาญ - ต้องการการฝึกฝนและความมุ่งมั่น

การควบคุมกระบวนการเชิงประจักษ์ (3 เสาหลัก):
1. ความโปร่งใส - ทุกด้านมองเห็นได้
2. การตรวจสอบ - ตรวจสอบความคืบหน้าบ่อยครั้ง
3. การปรับตัว - ปรับเปลี่ยนตามผลการตรวจสอบ
```

## **3. หัวข้อหลัก / Main Topics**

### **English Version**
```
MAIN TOPICS
===========

I. SCRUM ROLES (3 Roles)
------------------------
1. Product Owner
   • Maximizes product value
   • Manages Product Backlog
   • Defines "Done" criteria

2. Scrum Master
   • Ensures Scrum is understood and enacted
   • Removes impediments
   • Facilitates Scrum events

3. Development Team
   • Self-organizing and cross-functional
   • Delivers "Done" increments
   • 3-9 members optimal

II. SCRUM EVENTS (4 Events)
---------------------------
1. Sprint Planning
   • What can be delivered?
   • How will the work be done?

2. Daily Scrum
   • 15-minute time-boxed
   • Plan for next 24 hours

3. Sprint Review
   • Inspect the Increment
   • Adapt Product Backlog

4. Sprint Retrospective
   • Plan improvements
   • Improve process

III. SCRUM ARTIFACTS (3 Artifacts)
----------------------------------
1. Product Backlog
   • Ordered list of requirements
   • Continuously refined

2. Sprint Backlog
   • Selected Product Backlog items
   • Plan for delivering Increment

3. Increment (with Burndown Chart)
   • Sum of completed Backlog items
   • Must be in usable condition
```

### **ภาษาไทย**
```
หัวข้อหลัก
=========

I. บทบาทในสกรัม (3 บทบาท)
------------------------
1. Product Owner
   • เพิ่มมูลค่าผลิตภัณฑ์สูงสุด
   • จัดการ Product Backlog
   • กำหนดเกณฑ์ "งานเสร็จ"

2. Scrum Master
   • ดูแลให้ใช้สกรัมถูกต้อง
   • แก้ไขอุปสรรค
   • อำนวยความสะดวกกิจกรรม

3. ทีมพัฒนา
   • จัดการตัวเองได้และมีหลายทักษะ
   • ส่งมอบงานที่ "เสร็จแล้ว"
   • ขนาด 3-9 คนเหมาะสมที่สุด

II. กิจกรรมในสกรัม (4 กิจกรรม)
-----------------------------
1. การวางแผน Sprint
   • จะส่งมอบอะไร?
   • จะทำงานอย่างไร?

2. ประชุมประจำวัน
   • 15 นาทีจำกัดเวลา
   • แผนงาน 24 ชั่วโมงถัดไป

3. ทบทวน Sprint
   • ตรวจสอบ Increment
   • ปรับ Product Backlog

4. สะท้อนกลับ Sprint
   • วางแผนการปรับปรุง
   • ปรับปรุงกระบวนการ

III. สิ่งส่งมอบในสกรัม (3 สิ่ง)
-----------------------------
1. Product Backlog
   • รายการความต้องการเรียงลำดับ
   • ปรับปรุงอย่างต่อเนื่อง

2. Sprint Backlog
   • รายการจาก Product Backlog
   • แผนการส่งมอบ Increment

3. Increment (พร้อม Burndown Chart)
   • ผลรวมของงานที่เสร็จแล้ว
   • ต้องพร้อมใช้งานได้
```

## **4. คู่มือ Scrum / Scrum Handbook**

### **English Version**
```
SCRUM HANDBOOK
==============

QUICK REFERENCE CARD

TIME-BOXES:
• Sprint: 1-4 weeks (consistent duration)
• Sprint Planning: 8 hours max (for 4-week Sprint)
• Daily Scrum: 15 minutes daily
• Sprint Review: 4 hours max (for 4-week Sprint)
• Sprint Retrospective: 3 hours max (for 4-week Sprint)

KEY METRICS:
• Velocity: Work completed per Sprint
• Burndown: Remaining work over time
• Capacity: Team's available hours

DOD (Definition of Done):
• Code complete and reviewed
• All tests passing
• Documentation updated
• Accepted by Product Owner
```

### **ภาษาไทย**
```
คู่มือสกรัม
==========

บัตรอ้างอิงด่วน

กรอบเวลา:
• Sprint: 1-4 สัปดาห์ (ระยะคงที่)
• วางแผน Sprint: ไม่เกิน 8 ชม. (สำหรับ 4 สัปดาห์)
• ประชุมประจำวัน: 15 นาที ทุกวัน
• ทบทวน Sprint: ไม่เกิน 4 ชม. (สำหรับ 4 สัปดาห์)
• สะท้อนกลับ Sprint: ไม่เกิน 3 ชม. (สำหรับ 4 สัปดาห์)

ตัวชี้วัดสำคัญ:
• ความเร็ว: งานเสร็จต่อ Sprint
• Burndown: งานคงเหลือตามเวลา
• ความสามารถ: ชั่วโมงที่ทีมมี

เกณฑ์งานเสร็จ:
• โค้ดเสร็จและตรวจสอบแล้ว
• ทดสอบผ่านทั้งหมด
• อัปเดตเอกสาร
• Product Owner ยอมรับ
```

```
สรุป
====

แพ็คเกจสกรัมแบบครบวงจรนี้ประกอบด้วย:

1. บทนำ - ภาพรวมและปรัชญา
2. นิยาม - แนวคิดและหลักการพื้นฐาน
3. หัวข้อหลัก - รายละเอียดโครงสร้าง 3-4-3
4. คู่มือ - บัตรอ้างอิงด่วน
5. ลำดับงาน - แผนภาพกระบวนการ
6. เทมเพลตรายการงาน - ไฟล์ Excel สำหรับติดตาม
7. เทมเพลตรายการตรวจสอบ - ไฟล์ Excel สำหรับควบคุมคุณภาพ

ทั้งหมดจัดทำเป็น 2 ภาษา (อังกฤษและไทย) 
เพื่อให้ทีมหลากหลายวัฒนธรรมนำไปใช้ได้ง่าย
```

---

**หมายเหตุ**: สำหรับไฟล์ Excel จริง สามารถสร้างตามโครงสร้างด้านบน โดยเพิ่มสูตรคำนวณอัตโนมัติ กราฟ และ conditional formatting เพื่อให้ใช้งานได้สะดวกยิ่งขึ้น 📊🚀
